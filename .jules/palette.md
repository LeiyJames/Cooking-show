## 2024-04-01 - Interactive Grouped Controls & Framer Motion
**Learning:** Found that custom increment/decrement controls for features like the Ingredient Calculator were missing the `role="group"` attribute connecting them to their label. More importantly, when visually disabling interactive `framer-motion` elements (e.g. `disabled:opacity-50`), their `whileHover` and `whileTap` animations still fired, providing misleading tactile feedback on a disabled control.
**Action:** When creating grouped icon-only button controls, always wrap them in `role="group"` with `aria-labelledby`, use `aria-live="polite"` for the dynamic text span, and conditionally clear framer-motion props (e.g. `whileHover={isDisabled ? {} : { scale: 1.05 }}`) to match visual disabled states with interaction states.
