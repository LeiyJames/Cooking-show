## 2024-06-25 - Polyfilling generic element accordions for keyboard accessibility
**Learning:** In this application, accordion-like interactive elements (such as recipe step toggles) are sometimes implemented using generic `<div>` tags instead of semantic `<button>` tags. This breaks keyboard navigation and screen reader support, as `<div>` elements do not natively receive focus or handle key events.
**Action:** When encountering interactive generic elements like `<div>` that expand/collapse content, fully polyfill them by adding `role="button"`, `tabIndex={0}`, an `onKeyDown` handler listening for 'Enter' and ' ' (Space) to trigger the toggle action, and the standard `aria-expanded` and `aria-controls` attributes linking to the expandable region's ID. Also ensure a visual `focus-visible` state is present.
