## 2024-03-06 - Accessible Icon Buttons and Dynamic State in Counters
**Learning:** Icon-only control buttons (like + / - / reset) in dynamic components like IngredientCalculator lacked ARIA labels and title tooltips, and the resulting count changes were not announced to screen readers. Additionally, disabled states (e.g., when counter is at 1) lacked visual indication or keyboard focus styling.
**Action:** When creating counter components or icon-only controls, always add `aria-label` and `title` to buttons. Ensure disabled buttons have `disabled:opacity-50 disabled:cursor-not-allowed` for clear visual disabled states and handle animation disablement. Finally, wrap dynamic text outputs (like a counter value) in an element with `aria-live="polite"` and `aria-atomic="true"` so value changes are automatically announced to assistive technologies.
