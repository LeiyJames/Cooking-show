## 2024-03-28 - Framer Motion Interactive Disabled States & Custom Form Controls
**Learning:** When using Framer Motion with visually disabled HTML attributes (e.g., `disabled:opacity-50 disabled:cursor-not-allowed`), interactive tactile feedback (like `whileHover` and `whileTap`) still triggers on disabled buttons. Additionally, grouping increment/decrement buttons effectively for screen readers requires treating the container as a group control.
**Action:** When adding visual disabled states to `framer-motion` elements, always conditionally disable their motion properties (e.g., `whileHover={isDisabled ? {} : { scale: 1.05 }}`) to ensure the element is visually AND interactively disabled. Also, for custom numeric steppers, apply `role="group"` and `aria-labelledby` on the wrapper, and use `aria-live="polite"` on the dynamic value display.
