## 2024-05-24 - [Fixing API Thrashing with Derived State]
**Learning:** In `app/components/TimerContext.tsx`, having long-lived browser APIs like `setInterval` and `navigator.wakeLock.request` inside `useEffect` hooks with state dependencies (like `[timers]`) that update every second causes severe API thrashing (clearing/resetting the interval and releasing/requesting the wake lock every 1000ms).
**Action:** Extract stable boolean derivations (e.g., `const hasRunningTimer = Object.values(timers).some(...)`) from the state during the render cycle, and use those derived primitive booleans as the `useEffect` dependencies instead of the full changing state object. This ensures the effect setup/teardown only occurs when the condition actually changes (e.g., from 0 running timers to 1 running timer).
