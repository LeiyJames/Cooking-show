## 2024-05-24 - TimerContext Interval Thrashing Optimization
**Learning:** Using an entire state object (like `timers`) that updates frequently as a dependency in a `useEffect` that sets up long-running APIs (like `setInterval` or `navigator.wakeLock`) causes those APIs to be constantly re-created. In this codebase, the countdown loop updated the `timers` object every second, causing the WakeLock and Interval to thrash once per second.
**Action:** Always derive stable primitive booleans (e.g., `hasRunningTimer`) outside the `useEffect` and use those in the dependency array instead. When using `setInterval`, use the functional update form of the state setter `setTimers(prev => ...)` to ensure the interval closure always has access to the latest state without needing the state object in its dependencies.
